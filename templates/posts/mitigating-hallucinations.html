Mitigating Hallucinations Using External API Validation

Published: Feb 2026 • 5 min read

The Problem LLMs fabricate structured data when uncertain. In a travel planning system, hallucinating: Flight numbers Prices Schedules Is unacceptable.

The Solution: Enforced External API Validation

Flight data sourced from: Amadeus IT Group Instead of: LLM generating flight options I enforced: LLM → Intent Extraction → API Call → Structured Validation → Response

Validation Architecture

LLM extracts structured query:

origin

destination

dates

API returns real data

System schema-validates response

LLM formats only validated data

The LLM is never allowed to invent flight details.

Guardrails Implemented

JSON schema enforcement

Strict field validation

Response checksum

No free-text generation for structured fields

Result

Hallucinated flight data dropped to zero.

User trust increased.
System reliability improved dramatically.

Core Insight

LLMs should interpret. They should not invent critical structured data.
